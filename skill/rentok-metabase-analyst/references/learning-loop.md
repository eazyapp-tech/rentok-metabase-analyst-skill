# Learning Loop

Use this when a teammate finds a client issue, workaround, stale dashboard, missing caveat, or corrected query.

## Rule

Learn from every real usage issue, but promote only reviewed learnings into the shared skill.

Do not silently auto-edit everyone else's skill from one local workaround.

## Capture Format

```markdown
# Learning Note

Date:
Author:
Client:
Team:

## User question

## What failed

## Working fix

## Evidence

## Confidence

## Proposed promotion
```

## Promote When

- the issue is likely to recur
- the fix is backed by evidence
- the fix preserves read-only safety
- the fix does not turn a heuristic into fake certainty
- at least one gold-pack or regression check still passes

## Do Not Promote

- secrets or credentials
- one-laptop hacks
- stale dashboard numbers
- guessed schema meaning
- unchecked finance formulas
